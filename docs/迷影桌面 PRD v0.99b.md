# 迷影桌面 PRD v0.99b

> 文档状态：Skill 发布候选版  
> 当前日期：2026-05-28  
> 产品形态：跨 agent 的电影海报 / 壁纸生成 skill  
> 当前结论：skill 进入对外发布候选状态；固定开场句，一轮完成基础设置确认，必须调用生图模型生成海报；无生图能力时只交付统一提示词。v0.99b 新增角色入镜确认与用户上传参考图优先流程。

## 1. 产品定义

迷影桌面 v0.99b 是一个可配置在各类 AI agent 中的技能。用户提供真实电影名，skill 用一轮对话确认尺寸、输出路径、文字模式和生图路径，并在生图前确认是否需要电影角色入镜；若需要，优先要求用户上传角色照片/剧照作为强参考图，然后调用生图模型生成原创电影海报/壁纸。默认用户只看到最终海报和文件路径。

一句话定义：

> 请问你想生成哪部电影的海报壁纸？

## 2. 核心变化

- 不内置“定时换桌面壁纸”功能。Windows 和 macOS 已有系统级定时换壁纸能力。
- 不默认抓取豆瓣或海报站现成海报。
- 不把抽象美感放在电影识别度之上。
- 重点从“桌面工具”转为“电影海报/壁纸生成向导”。
- 开场引导语固定且只能有一句：`请问你想生成哪部电影的海报壁纸？`
- 基础设置必须一轮确认：尺寸/设备、保存位置、文字版本、生图方式、是否记忆默认设置。
- 必须根据影片自身影调决定亮暗、色彩、对比度和材质，不默认阴暗。
- 必须通过真实角色视觉锚点增强识别度，避免反复生成远景背影小人。
- 支持在用户确认后记忆默认尺寸和输出目录。
- 默认最终交付只包含海报/壁纸文件和极简状态，不展开研究报告、视觉 brief、manifest、提示词或负面提示词。
- 提示词包和研究信息仍保存到文件，供复现和调试；只有用户要求、prompt-only 模式或失败排查时才展示。
- Skill 运行应节省 token：少解释、少中间汇报、少文档输出，优先完成图片。
- 必须调用生图模型生成海报；不得用 HTML/CSS 生成或合成海报。
- 如果所在 agent 没有生图能力，固定提醒：`您的agent没有生图能力，我可以为您提供对应的提示词，您可以自行在其他生图模型中使用。`
- 提示词必须是一个统一大提示词，不拆分正向提示词、负向提示词和文字排版提示词。

## 3. 用户引导

加载 skill 后，第一句话必须固定：

```text
请问你想生成哪部电影的海报壁纸？
```

之后尽量集中确认：

1. 电影身份  
   如果电影名有歧义，列出候选并请用户确认。

2. 一轮基础设置  
   电影确认后，必须把尺寸/设备、保存位置、文字版本、生图方式、是否记忆默认设置集中在一轮对话中询问，不拆成多轮。已有默认值时要展示默认值，允许用户沿用。问题必须分行编号，不得用一整段文字糊在一起。

   标准模板：

   ```text
   请一次确认下面 5 项设置：
   1. 您想生成的海报/壁纸尺寸是多少？如果不知道具体分辨率，也可以告诉我目标设备型号或“当前电脑桌面”。
   2. 文件生成后想保存到哪个本地文件夹？如果沿用默认位置，请直接说“默认”。
   3. 您想要哪种版本：带文字、无文字，还是两者都要？
   4. 生图方式选择哪一种：使用当前 agent 的生图能力、外部 API/生图工具，还是只要提示词？
   5. 是否把以上四项都记为默认设置，方便下次沿用？
   ```

   四项基础设置都支持记忆默认：尺寸/设备、输出位置、是否带文字、生图形式。下一次使用时必须展示之前的默认值，并允许用户直接说“全部默认”；如果用户只修改其中一项，其余设置继续沿用默认。

3. 目标尺寸  
   支持：
   - 明确尺寸，如 `3840x2160`；
   - 当前设备屏幕；
   - 手机/平板/电脑型号；
   - 通用比例，如 `16:9`、`9:16`、`4:5`。

4. 设备分辨率查询  
   如果用户要做手机壁纸但不知道尺寸，请用户提供具体型号，skill 联网查询分辨率，并在生成前让用户确认或沿用该尺寸。

5. 输出目录  
   在一轮基础设置中确认保存位置。若用户选择记忆默认设置，后续对话可显示并沿用该保存位置。

6. 文字版本  
   在一轮基础设置中确认：
   - 带文字；
   - 无文字；
   - 两者都要。

   带文字版本必须由生图模型直接生成完整海报文字排版。不得用 HTML/CSS 或后期网页排版生成文字海报。

7. 生图方式  
   在一轮基础设置中确认：当前 agent 生图能力、外部 API、其他生图工具，或只要提示词。若 agent 无生图能力，按固定话术提示用户并交付统一提示词。

8. 角色入镜确认  
   生图前必须单独确认用户是否希望海报中出现电影角色形象。标准话术：

   ```text
   是否希望海报中出现电影角色形象？如果希望，请在对话里上传一张你希望出现在海报中的角色照片/剧照（清晰正脸、侧脸或半身优先）；如果不希望，请说“不出现人物”。
   ```

   如果用户选择出现人物，用户上传的角色照片/剧照是第一优先级参考图。没有可附加到生图模型的参考图时，不得生成可识别人物脸，也不得承诺精准还原；但仍可以用克制、巧妙的非露脸人物策略继续设计，例如背影、剪影、局部、手部、服装姿态、影子、倒影、小比例人物或人物与物件/空间融合。不要刻意把人物完全排除，除非海报概念本身更适合无人。

## 4. 视觉策略

默认生成“电影海报感”的原创视觉，而不是写实剧照。

所有生成图都应是美术风格海报，不使用真人写实风格。可使用油画、素描、水粉、丙烯、水彩、版画、现代主义、印象派、野兽派、抽象主义、波普艺术、构成主义、拼贴等美术/平面设计语言。

v0.8 起，视觉风格不应停留在“正常插画”或常规电影概念图。Skill 必须具备更大的艺术史与当代艺术调度空间。

v0.99a 起，默认风格选择改为随机抽取：影片理解负责决定海报中出现的元素、人物、关键道具、影调线索和中心隐喻；美术风格默认从风格池随机抽取。除非用户明确指定风格，否则不要让 agent 因为“更适合影片”而自动改回安全、匹配、常规的电影插画风格。

v0.99b 起，风格选择改为带权重随机，并提高经典美术风格权重。印象派/后印象派/点彩、立体主义、野兽派/表现主义、抽象/至上主义/构成主义、达达/超现实、波普、水墨、浮世绘、中世纪圣像/彩窗、文艺复兴/巴洛克、版画等不应是低频偶发项，而应在默认生成中经常出现。

v0.99a 起，新增视觉密度控制：每次生成必须选择 `visual_density`。海报可以热闹丰富，也可以简约、留白、点睛一笔。不得因为研究到很多剧情元素就把画面铺满。

如果用户反馈生成结果仍然常规、插画化、太安全，下一次生成必须启用 `style_escalation`，通过更激进的艺术语言、抽象机制和材料约束改变图像结构，而不是简单增加“更艺术、更大胆”等形容词。

### 4.1 艺术提炼

Skill 应先完成：

1. 剧情事实；
2. 情绪压力；
3. 中心隐喻；
4. 海报装置；
5. 艺术语言；
6. 影片影调；
7. 角色锚点；
8. 关键道具/物件锚点；
9. 视觉密度。

### 4.1.1 艺术语言系统

Skill 应为每次生成选择：

- `style_selection_mode`：`weighted_random`、`corrective` 或 `user_specified`；
- 一个明确的 `style_lane`；
- 一个具体的 `style_variant`；
- 一个主艺术语言；
- 一个辅助材料或视觉装置；
- 一个抽象机制；
- 一组能指/所指层次；
- 一个风格选择理由。

不要简单堆叠流派名。抽中的风格必须真正控制构图、质感、色彩和抽象机制；如果风格与影片形成反差，必须写出 `counterpoint_bridge`，说明反差如何揭示影片，而不是替换掉随机风格。

默认不能把 `fine-art poster`、`painterly`、`cinematic`、`beautiful illustration`、`movie poster style` 当成主风格。它们只能作为辅助描述，不能作为风格系统。

每次用户请求生成时都应重新生成，不得把之前同一影片的成品当缓存直接交付。默认风格选择应基于带权重随机抽取、用户偏好和当前视觉问题，而不是复用旧结果或由剧情自动匹配。

默认风格权重应向经典美术/艺术史语言倾斜。推荐基准：`impressionist_light_field`、`fragmented_modernism`、`expressive_color`、`geometric_avant_garde`、`conceptual_dada_surreal`、`east_asian_ink`、`renaissance_baroque_allegory`、`medieval_icon_glass`、`ukiyo_e_flatworld`、`print_process`、`pop_repetition_media` 合计应占主要概率。`real_object_still_life` 不进入普通随机池，除非海报概念确实围绕真实关键道具展开。

`style_lane` 可选：

- `geometric_avant_garde`：构成主义、至上主义、包豪斯、风格派、瑞士网格；
- `fragmented_modernism`：立体主义、未来主义、多视角断裂、分析拼贴；
- `expressive_color`：野兽派、表现主义、新表现主义、非自然情绪色彩；
- `pop_repetition_media`：波普、丝网印刷、半调、重复头像/物件、消费图像批评；
- `conceptual_dada_surreal`：达达、超现实、观念置换、不可能物件；
- `optical_digital_signal`：欧普艺术、动力网格、CRT、VHS、数字故障、信号噪声；
- `material_arte_povera`：灰烬、布、金属、纸、绳、尘土、现成物装置；
- `east_asian_ink`：中国水墨、文人画、日本画、笔触、留白和纸性；
- `gongbi_miniature`：工笔、波斯/印度细密画、矿物色、密集符号；
- `ukiyo_e_flatworld`：浮世绘、平面色块、织物纹样、季节性构图；
- `medieval_icon_glass`：中世纪手抄本、拜占庭圣像、哥特彩窗、金色平面；
- `renaissance_baroque_allegory`：湿壁画、古典寓言、巴洛克光线、舞台式人物；
- `social_mural_realism`：列宾派、墨西哥壁画、苏联宣传画、公共历史叙事；
- `print_process`：木刻、亚麻油毡版画、铜版蚀刻、石版画、risograph、丝网印刷；
- `textile_tapestry_map`：挂毯、刺绣、编织网格、地图/图解结构。
- `impressionist_light_field`：印象派、后印象派、点彩、破碎色彩、空气感笔触和光场结构；
- `minimalist_reduction`：极简主义、硬边绘画、稀疏几何、单一物件、克制留白和尺度张力。
- `real_object_still_life`：写实/真实/现实物件静物，强调电影关键道具的真实材质、形制、比例和光线。

每个 `style_lane` 还必须抽取具体 `style_variant`，例如：

- 印象派破碎光色、后印象派方向性笔触、点彩颗粒；
- 毕加索/布拉克式立体主义破碎平面、分析立体主义、多视角拼贴；
- 中国水墨写意、文人画留白、泼墨抽象、禅意水墨；
- 极简主义硬边色块、稀疏网格、单物件极简海报；
- 写实物件静物、真实道具特写、博物馆式物件研究、产品摄影式光线；
- 中世纪手抄本金地、哥特彩窗、拜占庭圣像；
- 木刻、蚀刻、丝网印刷、risograph、挂毯、刺绣等材料/工艺变体。

提示词必须让 `style_lane` 控制构图、材质和抽象机制，而不是把风格当装饰性形容词。

### 4.1.2 视觉密度系统

Skill 应为每次生成选择：

- `visual_density.mode`：`dense`、`balanced`、`sparse` 或 `single_stroke`；
- `selection_mode`：`user_specified`、`weighted_random` 或 `corrective`；
- `density_weights`：四种密度的最终权重；
- `density_inputs`：影片类型、规模、节奏、叙事复杂度、作者属性和视觉世界；
- 一个明确第一视觉焦点；
- 负空间/留白的作用；
- 最大主元素数量；
- 细节预算；
- 安静区域比例；
- 构图层级说明。

四种密度模式：

- `dense`：多符号、多层纹理、壁画/拼贴/挂毯逻辑。只能在风格或影片确实需要丰富性时使用，且必须有清晰主次。
- `balanced`：一个主视觉对象，一到两个辅助符号，有可呼吸的留白，是稳健的默认设计密度。
- `sparse`：一个主符号，大面积有意图的安静区域，少量辅助肌理，适合优雅、孤独、悬疑、诗性或图标化海报。
- `single_stroke`：一个决定性物件、手势、笔触、脸部局部或图形动作，极高留白，强调“点睛一笔”。

密度选择不能纯随机。默认使用带权重随机：先根据影片类型、规模、节奏、叙事复杂度、作者属性和视觉世界推导权重，再从权重中抽取结果。如果用户反馈画面太满、太累、抓不住重点，则下一次必须用 `corrective`，优先选择 `balanced`、`sparse` 或 `single_stroke`。

权重不是确定性规则，必须保留少量反向可能性。例如商业动作片更容易抽到 `dense`，但也可能抽到 `sparse` 的图标式反差海报；大师经典艺术电影更容易抽到 `sparse` 或 `single_stroke`，但也可能抽到 `balanced` 或少量 `dense`。

留白不是为了后期放文字的死白。好的留白必须产生焦点、节奏、呼吸、象征性沉默或尺度张力。坏的留白是明显空出来的标题框、死区或未完成区域。

如果选择 `sparse` 或 `single_stroke`，主元素最多一到两个，不得继续加入背景剧情碎片、装饰粒子、额外人物和多个道具。如果选择 `dense`，也必须写清楚前景焦点、次级节奏和背景肌理，避免全画面等权重堆满。

建议密度权重：

```json
{
  "commercial_action": {"dense": 45, "balanced": 35, "sparse": 15, "single_stroke": 5},
  "martial_arts_or_wuxia": {"dense": 35, "balanced": 35, "sparse": 20, "single_stroke": 10},
  "art_cinema_classic": {"dense": 10, "balanced": 30, "sparse": 40, "single_stroke": 20},
  "science_fiction_austere": {"dense": 10, "balanced": 30, "sparse": 40, "single_stroke": 20},
  "science_fiction_spectacle": {"dense": 30, "balanced": 35, "sparse": 25, "single_stroke": 10},
  "psychological_or_horror": {"dense": 15, "balanced": 30, "sparse": 35, "single_stroke": 20},
  "historical_epic_or_social_panorama": {"dense": 40, "balanced": 35, "sparse": 15, "single_stroke": 10},
  "quiet_drama_or_romance": {"dense": 10, "balanced": 40, "sparse": 35, "single_stroke": 15}
}
```

如果影片同时符合多个类别，应混合最接近的两到三个分布，再抽取一次。

所有密度分布完成后，还必须整体提高极简视觉出现概率：将 `sparse` 与 `single_stroke` 的合计权重提高 40%，再归一化到 100。记录 `minimalism_boost_applied: true`。

### 4.1.3 写实物件风格

`real_object_still_life` 用于呈现电影中的关键实物道具，不用于人物露脸肖像。该风格中的“写实/真实/现实”指物件材质、形制、比例、磨损、使用痕迹和光线真实。

适用例子：

- 徐浩峰《师父》中的咏春八斩刀；
- 《老无所依》中杀手使用的高压喷气枪/栓枪；
- 《穿普拉达的女王》中的高跟鞋；
- 《哈利·波特》中的魔杖；
- 也可用于戒指、钥匙、手袋、车门、录音带、面具、相机、文件等强识别道具。

限制：

- 必须先建立关键道具身份锁定；
- 不得用泛化道具替代电影中具体道具；
- 人物不能出现可识别面部；
- 可以出现身体局部，例如手、脚、袖口、肩部、背影边缘、影子；
- 画面仍然是海报设计，不是普通商品广告。

- 未来主义；
- 野兽派；
- 达达；
- 超现实主义；
- 包豪斯；
- 风格派；
- 波普艺术；
- 欧普艺术；
- 极简主义；
- 观念艺术；
- 新表现主义；
- 贫穷艺术；
- 粗野主义平面设计；

- 印度细密画；
- 新艺术运动；
- 装饰艺术；
- 苏联宣传画；
- 墨西哥壁画。

材料与工艺：

- 木刻；
- 石版画；
- 丝网印刷；
- risograph；
- 铜版蚀刻；
- 撕纸拼贴；
- 织物/挂毯；
- 陶瓷釉色；
- 金箔；
- 木炭与粉彩；
- 湿壁画墙面；
- 彩色玻璃；
- 蓝图/技术图纸；
- 档案复印；
- 热敏打印；
- VHS / CRT 影像噪声。

可吸取经典与当代艺术家的设计语法，例如安迪·沃霍尔式的重复图像与消费符号、大卫·霍克尼式的明亮空间与拼贴视角、罗斯科式色域压力、康定斯基式音乐抽象、马列维奇式几何还原、毕加索/布拉克式立体主义破碎、马蒂斯式野兽派色彩、杜尚式观念置换、博伊斯式材料仪式、草间弥生式重复图案、白南准式视频信号等。

涉及在世艺术家时，不直接要求“模仿某某风格”，而是蒸馏其可观察的视觉操作：色彩、构图、材料、重复、符号、空间、节奏、观念机制。

### 4.1.2 反差风格

Skill 可以有意识地使用反差风格，但必须服务影片主题。

示例：

- 科幻电影可转译为中国水墨、浮世绘、中世纪手抄本或文艺复兴寓言；
- 超级英雄电影可转译为构成主义几何、波普系统、湿壁画或中世纪战争挂毯；
- 社会现实电影可转译为工笔静物、贫穷艺术材料或观念艺术装置；
- 恐怖片可转译为极简抽象、彩窗或冷峻包豪斯网格；
- 爱情片可转译为立体主义双重视角、色域绘画或浮世绘季节构图。

反差风格必须写出桥接理由：

```text
虽然这部电影属于 [类型/时代]，但 [艺术语言] 可以通过 [视觉机制] 揭示 [主题/情绪/结构]。
```

不能把文化风格当装饰，也不能为了猎奇牺牲电影识别度。

### 4.1.3 风格升级机制

当输出像普通插画、概念图或安全电影海报时，skill 必须进行结构性升级：

1. 从常规海报设计升级为明确艺术流派；
2. 从风格标签升级为抽象机制；
3. 从表面纹理升级为材料/工艺约束；
4. 从好看升级为能指/所指层次；
5. 写明该艺术语言为什么揭示这部电影。

以《色，戒》为例，不应只生成优雅民国时期插画。可考虑：

- 麻将立体主义：脸、手、旗袍、麻将牌、监视空间被切成多视角平面；
- 工笔间谍细密画：丝绸、匕首、戒指、窗格与压抑情欲形成精密秩序；
- 沃霍尔式重复身份：重复面具、口红、红印章和商业色彩被转化为焦虑；
- 构成主义欲望陷阱：红黑斜线、身体向量、政治几何和隐秘代码；
- 水墨 noir 反差：身体消融为墨迹，戒指或烟头成为唯一饱和符号。

### 4.2 影片影调

Skill 必须先判断影片自身的影像语言，再决定壁纸的亮暗、色彩、对比度和材质，不应把“阴暗”“低饱和”“黑底”当成严肃电影的默认答案。

需要研究并记录：

- 彩色 / 黑白 / 混合 / 动画；
- 高调、中调、低调或混合影调；
- 对比度：柔和、硬朗、扁平、明暗法、图形化等；
- 色温：暖、冷、中性或混合；
- 胶片、数字、动画、纪录片、档案影像等媒介线索；
- 摄影师或影片公认视觉风格；
- 影片气质与类型：青春、喜剧、爱情、日常、宗教性、都市疏离、动作奇观、悲剧、荒诞等。

可选影调策略包括：

- 明亮暖调；
- 日常自然色；
- 粉彩忧郁；
- 日晒褪色；
- 黑白图形；
- 土色圣像；
- 高饱和波普；
- 霓虹/金属奇观；
- 发光的精神性；
- 冷调制度空间；
- 夜曲低调。

低调、黑底、阴郁影调只在影片本身需要时使用。比如动作奇观、青春喜剧、暖色日常片、彩色动画、明亮外景片，不应被统一压成暗调。

### 4.3 角色锚点

如果生成图与电影联系不强，必须加入主人公或关键人物视觉锚点。

角色锚点不应只是一种远景背影。应支持多种呈现：

- 近景符号肖像；
- 侧脸或四分之三侧脸；
- 剪影；
- 背影；
- 人物局部；
- 服装 + 道具；
- 主配角组合；
- 男/女主关系构图；
- 对立角色 split composition；
- 群像星座式构图；
- 人物与物件/空间融合；
- 光影中的环境肖像。

Skill 应搜索并提取主人公、主演或关键角色的视觉特征：

- 面部结构或关键五官特征；
- 轮廓；
- 服装；
- 发型；
- 姿态；
- 道具；
- 色彩；
- 空间；
- 重复动作或标志性表情。

不要直接复制剧照、宣传照或官方海报。

一旦呈现电影人物，必须严格契合电影中的真实角色形象。可以让整体海报采用美术风格，但人物身份不能只靠文字描述或“风格化近似”来猜。不能捏造一个电影中不存在的角色形象，也不能用泛化的帅哥/美女替代真实角色。

重要区别：非真人写实不等于文字近似。Skill 禁止的是照片级写实、照搬剧照、明星写真式肖像；真正目标是参考图身份保持。只要画面出现由真实演员/表演者塑造的角色，就必须先获取电影中的真实剧照/截图图片文件，必要时裁出人物脸部/半身参考图，并把这些图片作为生图参考输入。不得说“避免真人脸部复刻”来规避角色还原目标。

参考图的用途是提取身份与外貌锚点，而不是把图中人物抠出来复制到海报中。Skill 必须保留人物脸型、发型、年龄感、表情气质、服装身份、姿态语言和角色气息，但要在新的海报构图、艺术语言、光色和材质中重新生成。不得抠图、贴图、描摹、克隆参考图，也不得复制参考图的姿势、背景、构图、光线、镜头裁切或画面角度，除非用户明确要求复刻某张剧照。

角色呈现应更大胆、更正面：当影片识别度需要人物时，可以使用正脸美术肖像、四分之三侧脸、半身像、人物关系双人构图或关键角色群像。背影和远景小人只能作为特定海报策略之一，不能成为默认模式。

### 4.3.1 角色身份锁定

只要海报中出现具体电影角色，skill 必须先建立角色身份锁定，而不是只写“男主”“女主”或演员姓名。

角色身份锁定包括：

- 角色名；
- 演员/表演者；
- 电影名与年份；
- 角色关系；
- 角色在片中的年龄感、气质和行为姿态；
- 必须保留的脸部、发型、服装、道具、表情、场景线索；
- 禁止替换项，例如泛化帅哥/美女、现代时装版本、无关演员脸、电影中不存在的新角色。

提示词必须同时写明角色身份和电影身份，例如：

```text
《色，戒》(2007) 中的王佳芝，而不是泛化的 1940 年代旗袍女性。
```

### 4.3.2 参考图优先

如果生图模型支持参考图输入，涉及具体人物时必须优先使用角色参考图。参考图必须是图片文件，不是 URL、文字描述或演员姓名。优先级：

1. 用户提供的电影剧照或截图；
2. 官方剧照或官方预告片截图；
3. 制作剧照；
4. 可信电影数据库图片；
5. 可靠媒体文章图片；
6. 演员普通照片只能在缺少角色剧照时辅助脸部结构，不能作为主要角色参考。

v0.99b 当前版本采用“用户上传优先”的第一版稳态流程：生图前先询问是否需要角色入镜；若用户希望出现电影角色，优先请用户上传其想要锚定的角色照片/剧照。用户上传图是本轮最权威的角色身份来源，不应被自动搜索结果替换。

用户上传图是身份参考，不是海报素材层。提示词必须明确：使用参考图提取长相、外貌、服装和角色气质；重新生成新的海报人物，不复制参考图本身。

如果用户没有上传、上传图质量不足，或需要额外服装/姿态/场景参考，并且当前 agent 具备联网、browser use、下载或视频截帧能力，skill 可以自动寻找并获取角色参考图作为补充。

自动获取参考图包括：

- 搜索角色在片中的官方剧照；
- 搜索官方预告片、官方片段或可访问视频中的角色画面；
- 搜索制作剧照；
- 搜索可信电影数据库或可靠媒体文章中的角色图片；
- 在环境允许时下载图片或截取预告片帧，作为本轮生图参考。

获取后必须建立参考图包：

- 原始剧照/截图文件；
- 至少一张清晰脸部/半身人物裁切图；
- 可选服装/姿态裁切图；
- 可选场景上下文图；
- 实际传入生图模型的参考图路径或附件记录。

推荐查询：

```text
[电影名] [年份] [角色名] still
[电影名] [年份] [演员名] [角色名]
[电影名] official still [角色名]
[电影名] trailer [角色名] screenshot
[原名] [角色名] film still
```

如果用户选择出现人物但没有提供可用角色图，skill 应说明：若要精准还原脸部，需要上传角色照片/剧照；如果用户不上传，则继续使用非脸部人物策略。当前 agent/模型不能附加参考图时，不得承诺人物脸部还原，但可以通过手、服装、道具、剪影、背影、局部、影子、倒影、物件身体融合或文字主导海报来保留人物气息。

不要把官方海报作为主要角色参考，以免生成结果复制现成海报。

自动获取必须遵守限制：不绕过登录墙、付费墙、验证码、反爬或版权保护；不把参考图当作最终输出素材直接拼贴；只作为本轮角色身份锚定或用户明确同意的本地参考库。

### 4.3.3 角色识别验收

生成后必须检查：

- 不看标题时，熟悉电影的人是否能认出这个角色；
- 是否使用了角色参考图；
- 如果用户要求人物入镜，是否先请求并使用用户上传的角色参考图；在上传不足时是否再自动补充；
- 如果没有使用参考图，是否记录了 `text_only` 风险；
- 如果没有参考图，是否避免可识别脸部，并用不露怯的非露脸人物策略保留电影人物气息；
- 是否出现了泛化人物脸、现代造型或不存在的角色；
- 角色是否服务海报隐喻，而不是只作为装饰人物。

如果角色不像，应优先重试，而不是直接接受：

1. 切换到参考图模式；
2. 减少人物数量，先锁定最重要的一两个角色；
3. 从全身改为脸部/半身；
4. 强化必须保留特征和禁止替换项；
5. 如果模型仍无法保留角色身份，改用非脸策略：手、服装、道具、剪影、物件身体融合或文字主导海报。

### 4.3.4 关键道具/物件身份锁定

只要海报中出现能影响影片识别的道具、武器、服装、车辆、建筑、器物、乐器、工艺物或其他文化/历史特定物件，skill 必须先建立物件身份锁定。不得用泛化物件替代。

物件身份锁定包括：

- 物件在电影中的名称与必要原文名；
- 真实类别、用途和文化/历史语境；
- 形状、比例、材质、颜色、握持/使用方式、磨损或装饰细节；
- 电影中谁使用、出现在哪类场景、承担什么象征功能；
- 可用视觉参考或可靠文字来源；
- 禁止替换项。

例如《师父》中的咏春八斩刀，必须先确认八斩刀/蝴蝶双刀的真实形制、握柄、短宽刀身、成对使用方式和咏春语境，不得生成成普通匕首、军刀、武士刀或奇幻武器。

如果无法找到可靠物件参考，应询问用户提供剧照/参考图，或降低该物件在画面中的识别权重。一个显眼但错误的关键道具，比不用这个道具更破坏沉浸感。

## 5. 文字策略

文字策略以图像模型直接生成完整海报为唯一主路径。带文字版本的文字、构图和排版必须一起进入生图提示词，由生图模型完成。

文字版本分为：

- `no_text`：无文字主视觉；
- `with_text`：由生图模型直接生成整张带文字海报；
- `both`：同时输出两版。

带文字版最多包含：

- 片名；
- 原名；
- 导演/年份；
- 国家/地区；
- 一句策展短语。

文字渲染模式：

- `model_text`：生图模型直接生成文字；
- `no_text`：无文字版；
- `both`：同时生成无文字版和带文字版。

如果模型生成文字出现错字、乱码、语种错误或不可读，应重试更短文字，或交付无文字版加统一提示词；不得切换到 HTML 生成。

带文字版不得改变整张海报的亮度、色彩、对比度、饱和度或影调。文字只能通过排版、文字阴影、描边、局部小底、局部微弱遮罩等方式保证可读性；禁止给整张海报加暗层、全局滤镜、全局渐变或整体降亮度。带文字版和无文字版应保持基本一致的整体视觉效果。

排版语言规则：

- 第一行：中文片名；
- 第二行：影片对应国家/语言的原名，例如英美电影用英文，法国电影用法文，德国电影用德文，日本电影用日文；
- 导演、国别都用中文；
- 策展短语默认中文，除非用户指定其他语言。

## 6. 输出结构

建议输出到用户确认的本地目录：

```text
CinephileWallpaper/
├── outputs/YYYY/MM/
│   ├── YYYY-MM-DD_HHMM_Title_visual.png
│   ├── YYYY-MM-DD_HHMM_Title_wallpaper_with_text.png
│   ├── YYYY-MM-DD_HHMM_Title_wallpaper_no_text.png
│   └── YYYY-MM-DD_HHMM_Title_manifest.json
└── settings/preferences.json
```

历史成品不得作为自动复用缓存。用户再次请求同一影片时，默认重新生成；只有用户明确要求“找回上次那张图/旧文件”时，才检索旧输出。

### 6.1 用户可见交付

默认最终回复只包含：

- 生成完成状态；
- 海报/壁纸文件路径；
- 如可渲染，直接展示生成图片；
- 若用户要求设置桌面，则返回设置结果。

默认不向用户展示：

- 研究报告；
- 长视觉 brief；
- 完整 manifest JSON；
- 完整 image prompt；
- 拆分的 negative prompt；
- model notes；
- 来源链接列表；
- 中间参考图清单。

这些内容可以保存到本地 manifest 或 prompt package 文件，但不占用最终回复 token。

只有以下情况才展示提示词包或研究信息：

- 用户选择 `prompt_only`；
- 没有可用生图能力；
- 生成失败，需要用户拿提示词去其他模型；
- 用户明确要求查看、复制或修改提示词；
- 用户要求调试某次失败或不满意的生成。

## 7. 版本演进

### v0.6

- 从 App 主体转向跨 agent skill。
- 确认不再以内置定时换壁纸为核心。
- 建立电影确认、尺寸确认、输出目录、文字版本、生图方式的基础流程。
- 当时要求每次返回提示词包；v0.9 起改为默认静默保存，按需展示。

### v0.8

- 新增影片影调研究：彩色/黑白、亮暗、对比度、色温、媒介质感和情绪类型。
- 新增影调反默认规则：严肃电影不自动等于阴暗、低饱和、黑底。
- 强化角色锚点：支持正脸美术肖像、四分之三侧脸、半身像、双人关系构图和关键角色群像。
- 强化角色准确性：涉及人物时必须契合电影中的真实角色形象，不能捏造不存在的角色。
- 新增角色身份锁定：出现具体角色时必须记录角色名、演员、片名年份、必要视觉特征和禁止替换项。
- 新增参考图优先：支持参考图的模型必须使用角色剧照/截图；不支持时记录 `text_only` 身份风险。
- 新增自动参考图获取：具备联网/browser/download/截帧能力时，skill 可自动寻找角色剧照或预告片帧补充用户上传不足的情况。
- 新增偏好记忆：尺寸和输出目录可在用户确认后设为默认。
- 新增风格升级机制：当输出仍然常规插画化时，必须启用结构性的 `style_escalation`。
- 新增风格路由器：每次生成必须选择具体 `style_lane`，并避免连续重复同类艺术语言。
- 新增直接文字路径：带文字版由图像模型直接生成，HTML/CSS 后期排版不再作为海报路径。
- 新增一轮设置：电影确认后集中询问尺寸、保存位置、文字版本、生图方式和默认记忆。

### v0.9

- 默认海报优先交付：最终回复只给图片/路径和简短状态。
- 提示词包、研究信息、manifest 默认保存不展示。
- 只有 prompt-only、失败、调试或用户明确要求时才展示提示词/研究信息。
- 强化 token 节省策略：减少中间说明、文档输出和重复确认。

### v0.99a

- 作为发布候选版。
- 固定开场引导语为 `请问你想生成哪部电影的海报壁纸？`，不得输出功能说明或示例。
- 恢复一轮基础设置确认，并支持记忆默认尺寸、保存位置、文字版本和生图方式。
- 禁止 HTML/CSS 海报生成或文字合成路径。
- 生图必须调用图像生成模型；没有生图能力时交付统一提示词。
- 提示词交付改为单个统一大提示词，不拆正向/负向/排版提示词。
- 桌面设置只在用户明确要求或已保存为偏好时执行，不再每次询问。
- 风格路由器和角色参考图闸门纳入发布验收。
- 新增角色入镜确认：生图前询问是否需要电影角色形象；若需要，优先要求用户上传角色照片/剧照并作为本轮强参考图。
- 风格选择默认改为带权重随机抽取：影片理解服务元素与隐喻，风格从美术史风格池中抽取；新增印象派光场、毕加索/布拉克式立体主义变体、中国水墨变体和极简主义。
- 新增真实物件静物风格；新增视觉密度 40% 极简提升；人物构图默认偏正脸、侧脸或斜侧脸，背影只作为少数有意策略。

### v0.99b

- 在 v0.99a 基础上新增角色入镜确认：生图前询问是否需要电影角色形象。
- 若用户选择角色入镜，优先要求上传角色照片/剧照，并把上传图作为本轮最权威的角色身份参考。
- 没有可附加到生图模型的角色参考图时，不得生成或承诺可识别人物脸部精准还原；但允许用背影、剪影、局部、服装姿态、影子、倒影、小比例人物等非露脸策略保留人物存在。
- 用户上传参考图只用于提取人物身份和外貌锚点，不得抠图、贴图、描摹或复刻参考图构图。
- 提高经典美术风格权重，避免默认生成长期落入常规插画或当代泛海报风。

## 8. 非目标

v0.99b 不做：

- 定时换桌面壁纸；
- 豆瓣海报自动抓取作为主路径；
- 账号系统；
- 云同步；
- 海报市场；
- 直接复制官方海报；
- 固定模仿某位在世设计师。
- 默认输出研究报告或长设计文档。

## 9. 验收标准

- 第一轮交互足够简洁，不做长篇功能介绍；
- 第一句话必须固定为 `请问你想生成哪部电影的海报壁纸？`；
- 能处理同名电影歧义；
- 能询问并确认壁纸/海报尺寸；
- 能把尺寸、保存位置、文字版本、生图方式和默认记忆集中到一轮设置里；
- 能在用户同意后记忆默认尺寸；
- 能根据设备型号联网查询分辨率；
- 能确认本地输出目录；
- 能在用户同意后记忆默认输出目录；
- 能询问带文字、无文字或两者都要；
- 生图前能询问是否希望出现电影角色形象；
- 用户希望出现角色时，能要求上传角色照片/剧照，并将其作为本轮优先参考图；
- 能记忆文字版本和生图方式，并在下次允许“全部默认”；
- 能在用户要求或 prompt-only 模式下返回可复制提示词；
- 成功生成图片时默认不展开提示词、研究报告或 manifest；
- 最终回复以图片/路径为主，token 占用低；
- 能避免所有海报都落入常规插画风格；
- 每次生成能记录并执行具体 `style_lane`；
- 未指定风格时能带权重随机抽取 `style_lane` 和 `style_variant`；
- 经典美术风格应有显著权重，而不是低频偶发项；
- 能生成印象派、立体主义、水墨、极简主义等经典美术风格，而不只是常规插画；
- 每次生成能通过带权重随机选择并记录 `visual_density`；
- 能根据影片类型、规模、节奏、叙事复杂度、作者属性和视觉世界调整密度权重；
- 能在密度权重上整体提升 `sparse` + `single_stroke` 的出现概率 40%；
- 能生成 `sparse` 和 `single_stroke` 这类留白、简约、点睛一笔式海报；
- 能区分设计性的负空间和为了文字预留的死白；
- 能避免全画面等权重铺满元素，保证第一视觉焦点清晰；
- 能选择现当代、古典、地域传统、材料工艺或反差风格作为艺术语言；
- 能在随机风格和影片内容之间建立 `counterpoint_bridge`；
- 能产生抽象感、意境感和能指/所指层次，而不是只做剧情插图；
- 用户反馈常规插画化后，下一次必须启用 `style_escalation`；
- 能根据影片自身影调决定明暗、色彩和材质，不把所有严肃影片压成阴暗低饱和；
- 能通过多样化人物锚点增强电影识别度；
- 涉及具体人物时能建立角色身份锁定；
- 支持参考图时能把角色剧照/截图裁切为人物参考图，并实际附加到生图调用；
- 用户上传的角色图必须优先于自动搜索图；
- 用户上传参考图只能作为身份/外貌参考，不得抠图贴图或复制参考图姿势、背景、光线和构图；
- 没有可附加参考图时，不得生成或承诺可识别人物脸部精准还原；
- 用户不提供真实演员/角色照片时，仍可出现人物，但必须是克制的非露脸策略，避免捏造泛化脸；
- 具备联网/browser 能力时，可在用户未上传或参考图不足时自动补充角色参考图文件；
- 能区分“禁止真人写实/复制剧照”和“必须用参考图做人物身份保持”；
- 不得以“避免真人脸部复刻”为理由生成不像演员/角色的泛化人物；
- 不反复生成“远景背影小人”；
- 能主动使用正脸、侧脸、斜侧脸、半身像、双人/群像等角色构图，并限制背影作为默认选择；
- 能使用 `real_object_still_life` 呈现真实关键道具，且人物不能露脸，只能出现手、脚、袖口、肩部等身体局部；
- 输出仍然是海报，不是写实剧照；
- 输出一律不是真人写实风格，而是美术/平面设计风格；
- 涉及人物时必须符合电影中实际角色形象，不捏造角色；
- 带文字版本遵守中文片名 + 原语种原名 + 中文导演/国别规则；
- 带文字版本必须直接由生图模型生成；
- 所有提示词交付为一个统一提示词；
- 无生图能力时必须提示用户 agent 无生图能力，并提供统一提示词；
- 不使用 HTML/CSS 生成或合成海报；
- 不为了后期文字刻意制造大面积留白，整体海报构图优先；
- 带文字版不得通过全局暗层、全局滤镜或整体降亮度来提升文字可读性；
- skill 文档不依赖 Codex 专属能力，其他 agent 可迁移使用。
